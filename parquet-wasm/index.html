It’s the GraalVM counterpart to this site’s Calcite Query Explainer (which takes the CheerpJ route); your file never leaves the page.

The Parquet reading is done by the same Java code that runs on a JVM — here it’s ahead-of-time compiled to a single .wasm module, so the program is the WebAssembly. Your file’s bytes are base64-encoded and handed to an exported Java function (globalThis.parquetInspect); the schema, metadata, and rows come back as JSON.

Needs a browser with WasmGC and exception handling (exnref) — recent Chrome or Firefox, no flags — and must be served over HTTP (see the README and colophon). The reader is single-threaded, since the wasm backend can’t compile Hardwood’s virtual-thread pipeline.

Java → WebAssembly via GraalVM Native Image. Runs entirely client-side — nothing leaves the page.
