A Java program reads an Apache Parquet file — the Hardwood parser decoding alltypes_plain.parquet from apache/parquet-testing — compiled to WebAssembly by GraalVM Native Image’s experimental wasm backend. It’s the GraalVM counterpart to this site’s Calcite Query Explainer, which takes the CheerpJ route to Java in the browser. Everything below is the program’s output, produced entirely in your browser.

What this is

        
The same Java code (Hardwood on the single-threaded reader path) runs identically on the JVM, under Node.js, and here in the browser as WebAssembly. The schema, metadata, and all eight rows come out byte-for-byte the same in every runtime.

        
The Parquet file is embedded in the wasm module as a classpath resource, since the wasm target has no filesystem. See the README and colophon; the Java and the build live under java/.

      

      

        
Requirements & notes

        
    
          
  • Needs a browser with WasmGC and exception handling (exnref) — recent Chrome or Firefox, no flags.
    • 
          
  • Served over HTTP (wasm can’t load from a file:// page). Any static server works; no Range support needed.
    • 
          
  • The GraalVM launcher writes Java’s System.out through console.log; this page mirrors that into the panel above.
    • 
          
  • Single-threaded: the wasm backend can’t compile Hardwood’s virtual-thread pipeline, so a synchronous reader is used.
